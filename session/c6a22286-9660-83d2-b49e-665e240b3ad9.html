Moving from Apache Spark to Google Cloud Dataflow

Proposal

So, you need a data processing framework, one that will scale as your data grows - which should you choose ...? It’s a good question with many aspects. We @Juno chose the most dominant one - Apache Spark (1.x), and after some time, we decided to move to Google Cloud Dataflow. Google Cloud Dataflow is a fully managed Big Data as a service. With Dataflow you can develop wide data processing patterns including ETL, both batch and stream computation with the same interfaces. It supports auto-scaling, multi-zone for isolation, and much more. In this talk I will go through the reasons for moving, and explain about Google Cloud Dataflow briefly.
