Lightning Talk (5 min.)

Adi Polak

Big data software engineer

How I tweaked spark listeners for better monitoring

Proposal

Many companies deal with the challenges of analyzing Big Data. One of the most used engines for this task is Apache Spark. In any serious deployment of Spark where performance is crucial, good monitoring is essential in order to maximize performance. While Spark has many advantages, monitoring can become a painful issue without knowing Spark’s internals. While writing code for Spark is easy, improving performance adds another layer of complexity, which makes it easy to produce bad code unfit for production. In this talk I will present how I was able to tweak Spark listeners and utilize them for better monitoring, resulting in increased reliability that will help you sleep better at night
